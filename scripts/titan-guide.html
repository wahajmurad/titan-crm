Chapter 1
Getting Started
System Requirements

Project TITAN is built on a modern web technology stack that requires specific runtime environments for both development and production use. Before you begin, ensure that your system meets the following prerequisites: Node.js version 18 or later must be installed on your machine. You will also need either npm or bun as your package manager, with bun being the recommended option for faster startup times. A SQLite-compatible environment is required for the database layer, which comes bundled with the Prisma ORM. For deployment, Vercel is the recommended hosting platform, though any Node.js-compatible server will work.

Installation Steps
1

Clone the repository from your GitHub remote. Open a terminal window and navigate to your preferred working directory, then run the git clone command with your repository URL. This will download all project files including the source code, configuration files, and the Prisma schema definitions.

2

Install dependencies by running npm install or bun install in the project root directory. This command reads the package.json file and downloads all required packages including Next.js 16, React 19, Prisma ORM, Framer Motion, React Flow, Recharts, shadcn/ui components, and the Zustand state management library. The postinstall hook will automatically run prisma generate to create the database client.

3

Set up the database by running npx prisma db push. This command creates the SQLite database file and applies the schema defined in prisma/schema.prisma. It creates tables for Users, Permissions, Businesses, Leads, Campaigns, Outreach records, Meetings, Workflows, and all supporting models. If you need to reset the database at any point, use npx prisma migrate reset.

4

Start the development server by running npm run dev or bun dev. The application will start on port 3000 by default. Open your browser and navigate to http://localhost:3000. On first launch, you will be presented with the Setup screen to create your admin account.

5

Create your admin account through the initial setup screen. Enter your full name, email address, and a secure password (minimum requirements apply). This first account is automatically assigned the OWNER role with full permissions across all modules. After submission, the system reloads and takes you to the login page where you can sign in with the credentials you just created.

Quick Login Tip: The system uses stateless JWT-based authentication. Your session is stored in an httpOnly cookie. If you get logged out, simply sign in again with your email and password. There is no session expiration in the current version, so your session persists until you manually sign out or clear your browser cookies.

Running in Production

To run TITAN in production mode, first build the application with npm run build. This creates an optimized bundle in the .next directory. Then start the server with npm run start, which runs the standalone server file. For Vercel deployment, simply connect your GitHub repository to Vercel and it will automatically detect the Next.js framework and configure the build settings. Ensure your environment variables (particularly DATABASE_URL) are configured in the Vercel project settings dashboard.

Chapter 2
System Overview

Project TITAN is a comprehensive AI-powered sales and outreach platform designed to automate and intelligence-enhance every stage of the B2B sales pipeline. The system is organized into four primary sections accessible through the left sidebar navigation: Main, Campaigns, Intelligence, and Operations. Each section contains multiple feature modules that work together to create a seamless workflow from lead discovery to deal closure.

The platform uses a single-page application architecture built on Next.js 16 with the App Router pattern. Navigation between views is handled client-side using Zustand state management, which means transitions are instant and do not require full page reloads. Each view is lazy-loaded using Next.js dynamic imports, ensuring that only the code needed for the current view is loaded into the browser, resulting in fast initial load times and efficient memory usage.

Database Models

The TITAN database consists of 16 interconnected models that store all application data. The core entities include User accounts with role-based permissions (OWNER or TEAM), Business records that serve as company profiles, and Lead records that track each prospect through a nine-stage sales pipeline. Campaigns group leads and track outreach performance metrics. Outreach records store individual email communications with status tracking from draft through replied. Meeting records manage scheduled interactions with leads. Additional models include WebsiteAudit for site analysis scores across 10 categories, CompanyIntel for AI-generated business intelligence, Workflow and WorkflowRun for automation, PersonalizationScore for email quality metrics, AIMemory for system learning, PromptTemplate for AI prompt management, LeadProvider for data source configuration, and CampaignAnalytics for daily performance tracking.

Navigation and UI

The sidebar navigation is organized into four collapsible sections. When the sidebar is expanded (260px width), you see full labels for each item along with section headers. When collapsed (72px width), only icons are displayed with tooltips appearing on hover. The sidebar can be toggled using the arrow button on its right edge. At the top of the main content area, a sticky header displays the current view name, a Command+K shortcut hint for the command palette, a search button, and a dark/light theme toggle. The notification bell shows unread count and a dropdown of recent notifications. The Command Palette (activated by pressing Cmd+K or Ctrl+K) provides quick navigation to any view or action across the system.

Chapter 3
Main Section Features

The Main section contains the four most frequently used modules in the TITAN platform. These form the core workflow: discovering leads, auditing their websites, managing the lead pipeline, and monitoring overall performance through the dashboard. Each module connects to the next, creating a natural progression from prospect identification to qualified opportunity.

Dashboard

The Dashboard is your central command center that provides a real-time overview of your entire sales operation. When you log in, this is the first screen you see. It displays key performance indicators including total leads discovered, leads qualified through the pipeline, audits completed, emails sent, reply rate percentage, meetings booked, estimated revenue potential, and active campaign count. The data refreshes automatically by polling the dashboard API endpoint, giving you a live view of your team's activity. Below the stats row, you will find recent activity feeds, pipeline stage distribution charts, and quick-action shortcuts to navigate to other sections. The dashboard uses the user's name to provide a personalized greeting at the top of the page.

AI Command Center

The AI Command Center is a powerful natural language interface that lets you control the entire TITAN system through conversational commands. Instead of navigating through multiple screens, you can type instructions like "discover 20 leads in the healthcare industry in New York" or "audit the website example.com" and the system will execute the appropriate actions. The command center supports multiple action types including lead discovery, website auditing, lead qualification, email generation, campaign creation, data analysis, business intelligence generation, workflow creation, and report generation. Each command is processed through an AI interpretation layer that extracts the intent, parameters, and required actions. Results are displayed inline with step-by-step progress indicators showing what the AI is doing in real time.

The interface features a chat-style layout with your commands appearing on the right and system responses on the left. Each response includes the action taken, the outcome, and any data generated. You can also use the command center to chain multiple actions together, for example discovering leads and then immediately auditing their websites in a single workflow. This makes it the fastest way to execute multi-step sales processes without leaving a single screen.

Lead Discovery

The Lead Discovery module is where you find new potential customers for your outreach campaigns. You can search for businesses by specifying criteria such as industry type, geographic location (country and city), business size category, and specific keywords or service offerings. The discovery engine supports multiple lead provider sources including Google Places, Yelp, custom APIs, Serper search, and web scraping. Each lead provider can be configured separately in the Lead Providers settings section with their own API credentials, authentication methods, and priority levels.

When you initiate a discovery search, the system queries the configured lead providers and returns a list of matching businesses. Each result includes the business name, website URL (if available), industry classification, location, contact information, and a preliminary relevance score. You can review the results, select which businesses to add as leads, and the system will automatically create Business records and corresponding Lead records in the database with the initial stage set to DISCOVERED. The module also supports bulk operations, allowing you to add multiple leads to a specific campaign at once.

Website Audit

The Website Audit module performs a comprehensive 10-category analysis of any target website. When you enter a URL and initiate an audit, the system evaluates the website across UI Design, User Experience, SEO Optimization, Performance, Accessibility, Mobile Responsiveness, Security, AI Readiness, Automation Potential, and Conversion Optimization. Each category receives a score from 0 to 100, along with a detailed text analysis of specific findings, issues identified, and actionable recommendations.

The audit results go beyond technical scoring by also generating business intelligence. This includes an executive summary of the website's current state, a list of specific problems found, growth opportunities identified, automation and AI adoption opportunities, and a tailored pitch strategy with talking points designed to help you craft compelling outreach. The audit data is stored in the WebsiteAudit model linked to the corresponding Business record, making it available for email personalization and campaign strategy. This module is particularly valuable because it transforms a simple website analysis into a complete sales enablement package that gives you meaningful talking points for every prospect.

Leads Management

The Leads module is the central hub for managing your sales pipeline. It displays all leads in a filterable, sortable table with columns for business name, industry, stage, temperature (HOT/WARM/COLD), qualification score, assigned team member, and creation date. You can filter leads by any combination of these fields, and the table updates in real time as you adjust the filters. Clicking on any lead row navigates to the Lead Detail view, which provides a comprehensive profile of that prospect.

The Lead Detail view shows the full business information, all audit scores and findings, AI-generated analysis, the complete outreach history with sent emails and replies, scheduled and completed meetings, and an activity timeline. You can update the lead's stage manually by clicking through the pipeline stages: Discovered, Audited, Qualified, Outreach Sent, Replied, Meeting Booked, Proposal Sent, Won, or Lost. Each stage transition is logged in the activity feed. The lead detail also displays the AI's recommended solution, identified pain points, and decision maker information when available. From this view, you can compose new outreach emails, schedule meetings, assign the lead to a team member, or add notes.

Chapter 4
Campaign Section Features

The Campaigns section provides tools for organizing, executing, and tracking your outreach efforts at scale. Instead of sending individual emails one at a time, campaigns allow you to group leads by industry, location, or service offering and manage their outreach collectively. This section includes the Campaigns management view, the Email Center for composing and managing messages, and the Inbox for tracking replies.

Campaigns

The Campaigns module lets you create and manage outreach campaigns that group related leads together. When creating a campaign, you specify the campaign name, target industry, geographic location (country and city), target business size, the service offering you want to promote, and a daily sending limit to control the pace of outreach. Each campaign tracks aggregate performance metrics including the number of leads added, emails sent, emails opened, replies received, meetings booked, and deals won, along with a running revenue total.

The campaign list view shows all campaigns with their current status (ACTIVE, PAUSED, or COMPLETED) and key metrics at a glance. You can pause or resume campaigns, adjust the daily sending limit, and view the campaign's analytics over time through the CampaignAnalytics data which tracks daily performance metrics. Campaigns can be assigned to specific team members (the owner), and leads within a campaign inherit the campaign's targeting parameters for consistent messaging. This module is essential for running systematic, measurable outreach programs rather than ad-hoc email sending.

Email Center

The Email Center is your command post for composing, scheduling, and managing outreach emails. It provides a rich composition interface where you can write personalized emails to individual leads or create template-based messages for campaign-wide distribution. The email editor supports subject line optimization, body text with personalization variables, and attachment of generated assets such as audit reports, HTML demo pages, or ROI calculations.

Each email created in the Email Center is stored as an Outreach record with status tracking through the pipeline: DRAFT, SENT, OPENED, REPLIED, BOUNCED, or FAILED. The system tracks when emails are sent, opened, and replied to, with timestamps recorded for each event. You can schedule emails to be sent at optimal times, set up follow-up sequences, and view the complete email history for any lead. The Email Center also provides analytics on email performance including open rates, reply rates, and bounce rates, helping you continuously improve your outreach messaging and timing strategies.

Inbox

The Inbox module aggregates all incoming replies and responses from your outreach efforts into a unified view. Instead of checking multiple email accounts, the Inbox shows all replies sorted by date with the most recent first. Each reply is linked to the original outreach email and the corresponding lead, giving you full context without switching between screens. The inbox supports filtering by read/unread status, lead, campaign, and date range. You can mark replies as read, archive them, or take direct action such as scheduling a meeting or sending a follow-up response. The notification bell in the header also provides a real-time count of unread inbox messages, ensuring you never miss a time-sensitive reply from a prospect.

Chapter 5
Intelligence Section Features

The Intelligence section houses TITAN's AI-powered capabilities that provide deep insights, expert knowledge, and strategic guidance for your sales operations. These modules leverage AI models to analyze data, generate content, and provide recommendations that would otherwise require significant manual research and expertise. Each intelligence module serves a distinct purpose in augmenting your sales team's capabilities.

AI Assistant

The AI Assistant is a conversational chat interface that provides on-demand help with any aspect of your sales workflow. You can ask questions about specific leads, request help drafting emails, get advice on sales strategies, or ask for analysis of your campaign performance. The assistant has access to your system data and can reference lead information, audit results, and campaign metrics in its responses. The chat interface supports a continuous conversation thread, so you can ask follow-up questions and refine your requests naturally.

The AI Assistant is always accessible from the sidebar and is one of the two modules (along with Dashboard) that every user can access regardless of their permission settings. It uses the AI API endpoint to process queries and generate responses. The interface includes quick-action suggestion chips that let you jump to common tasks such as "Analyze my campaign performance," "Draft a follow-up email," or "Which leads should I prioritize?" This makes it an indispensable tool for both experienced sales professionals and new team members who are learning the system.

AI Agents

The AI Agents module provides a gallery of specialized AI agents, each designed to handle a specific aspect of the sales and outreach process. Each agent has its own configuration including system prompt, input schema, output format, temperature setting, and token limit. The available agent types include Lead Discovery Agent for finding prospects, Research Agent for deep company analysis, Website Intelligence Agent for automated site evaluation, Business Intelligence Agent for generating company profiles, Industry Expert Agent for domain-specific insights, Solution Architect Agent for designing service proposals, Offer Generator Agent for creating pricing packages, Personalization Agent for tailoring outreach content, Campaign Strategy Agent for planning outreach campaigns, Outreach Agent for generating email copy, and Learning Agent for improving system performance over time.

Each agent card in the gallery displays its name, description, configuration details, and a status indicator showing whether it is available for use. You can view and modify an agent's configuration, test it with sample inputs, and integrate it into your workflows through the Workflow Builder. The AI Agents module serves as the control center for managing TITAN's autonomous capabilities, allowing you to fine-tune each agent's behavior to match your specific industry and sales methodology.

Industry Expert

The Industry Expert module provides deep, domain-specific knowledge and insights for any industry vertical you are targeting. Instead of relying on generic sales advice, this module gives you access to AI-curated industry intelligence including market trends, common pain points, regulatory considerations, technology adoption patterns, and competitive landscapes specific to the industry you select. This information is invaluable for crafting outreach messages that demonstrate genuine understanding of a prospect's industry challenges.

When you select an industry, the module generates a comprehensive briefing that covers the current state of the industry, key terminology and jargon used by professionals in that space, typical budget ranges for services like yours, the decision-making process and common stakeholders involved, and seasonal patterns that might affect buying timing. You can use this intelligence to tailor your website audit findings, email messaging, and sales proposals to resonate with prospects in specific industries, dramatically increasing your credibility and response rates.

Strategy AI

The Strategy AI module is your strategic planning companion that helps you design and refine your overall sales approach. It analyzes your existing campaign data, lead pipeline metrics, and outreach performance to provide actionable recommendations for improving conversion rates at each pipeline stage. The module can identify bottlenecks in your sales process, suggest optimal lead qualification criteria, recommend the best times to send emails based on your historical open rates, and propose A/B testing strategies for your outreach messaging.

The Strategy AI interface presents its findings through a combination of data visualizations and narrative recommendations. You will see charts showing your conversion funnel from discovery to closed deal, comparisons of performance across different industries or geographies, and trend analysis of your key metrics over time. Each recommendation includes the reasoning behind it, the expected impact, and the specific actions you should take. This module transforms raw data into strategic insights, helping you make data-driven decisions about where to focus your sales efforts for maximum return on investment.

Prompt Library

The Prompt Library is a centralized repository for managing all the AI prompts used throughout the TITAN system. Prompts are organized into seven categories: Website Audit prompts for generating audit analyses, Industry Research prompts for gathering market intelligence, Lead Qualification prompts for scoring and prioritizing prospects, Email Generation prompts for crafting outreach messages, Sales Strategy prompts for planning approaches, Proposal Writing prompts for creating service proposals, and Campaign Analysis prompts for evaluating performance. Each prompt template has a name, category, the full prompt text, and a flag indicating whether it is the default prompt for its category.

You can create new prompt templates, edit existing ones, and set any prompt as the default for its category. The default prompt is automatically used by the corresponding AI module when no custom prompt is specified. This system allows you to continuously refine your AI interactions by improving the prompts based on real-world results. For example, if you notice that email generation is producing messages that are too generic, you can edit the Email Generation prompt to include more specific instructions about tone, personalization depth, and call-to-action placement. Changes to prompts take effect immediately across all modules that use them.

Chapter 6
Operations Section Features

The Operations section contains the tools and settings that keep your TITAN system running smoothly. This includes the advanced Hyper-Personalization engine for crafting highly targeted outreach, the visual Workflow Builder for automating multi-step processes, the Meetings scheduler for managing prospect interactions, the Lead Providers configuration for connecting external data sources, the Team management interface for user administration, and the Settings panel for system-wide configuration.

Hyper-Personalization

The Hyper-Personalization module is TITAN's most advanced outreach capability. It goes beyond simple mail-merge by running a comprehensive multi-stage pipeline that gathers and synthesizes data from multiple sources to create deeply personalized outreach content. The pipeline progresses through several stages: Research (gathering company data and market context), Audit (analyzing the prospect's website across 10 categories), Industry Analysis (understanding the competitive landscape and market position), Competitor Research (identifying who the prospect competes with), Problem Identification (determining pain points and challenges), Opportunity Mapping (finding areas where your services can add value), Solution Design (crafting tailored service recommendations), Sales Angle Generation (creating multiple persuasive approaches), Outreach Creation (writing the actual email content), and Quality Scoring (evaluating the personalization depth of the final output).

The quality scoring system evaluates each personalized email across six dimensions: Specificity (how much company-specific detail is included), Relevance (how well the content matches the prospect's needs), Value (how clearly the benefits are communicated), Tone (how appropriate the writing style is for the audience), CTA Quality (how compelling the call to action is), and an Overall score that combines all dimensions. Each score ranges from 0 to 100, and the system provides specific improvement suggestions for any dimension that scores below the threshold. The module also maintains AI Memory entries that store learnings about what personalization approaches work best for different industries and lead types, continuously improving its output quality over time.

Workflow Builder

The Workflow Builder provides a visual, drag-and-drop interface for creating automated multi-step processes. Built on React Flow v12, it displays a canvas where you can add, connect, and configure workflow nodes. Each node represents an action or condition in your automation pipeline, such as "Discover Leads," "Run Website Audit," "Generate Personalized Email," "Check Quality Score," "Send Email," or "Schedule Meeting." Nodes are connected by edges that define the flow of execution, and the canvas supports zooming, panning, and a minimap for navigating complex workflows.

Workflows can be triggered in four ways: manually by clicking a run button, on a schedule (cron-style timing), via webhook (external system triggers), or by system events (such as a new lead being added or an email being replied to). Each workflow maintains a version history and tracks execution statistics including total runs, error count, and the timestamp of the last run. The WorkflowRun model captures detailed execution data including the current node being processed, input and output data in JSON format, error messages if any, and the total execution duration in milliseconds. This makes it possible to debug failed workflows and optimize performance. The Workflow Builder is the key to scaling your sales operations without proportionally increasing manual effort.

Meetings

The Meetings module manages all scheduled interactions with your prospects and customers. It displays a calendar-style view of upcoming meetings along with a list view showing meeting details including the associated lead, meeting title and description, scheduled date and time, duration (default 30 minutes), status (SCHEDULED, COMPLETED, CANCELLED, or NO_SHOW), location or meeting link, and any preparatory notes. You can create new meetings, update existing ones, mark meetings as completed or cancelled, and add post-meeting notes. Each meeting is linked to a specific lead, so all meeting history is accessible from the Lead Detail view as well. The meetings module integrates with the notification system to send reminders about upcoming meetings, helping ensure that no prospect interaction falls through the cracks.

Lead Providers

The Lead Providers module configures the external data sources that power the Lead Discovery engine. TITAN supports five provider types: Google Places (for local business search), Yelp (for business reviews and listings), Custom APIs (for proprietary data sources), Serper (for web search results), and Web Scraping (for extracting data from specific websites). Each provider configuration includes the provider name, type, API URL, authentication credentials (API key, auth type, headers, and query parameters), active status, default provider flag, priority ranking, and usage statistics tracking success and failure counts.

You can add multiple providers of the same type with different configurations, for example two different Google Places API keys for different regions. The priority field determines the order in which providers are queried during lead discovery. Setting a provider as default means it will be used automatically when no specific provider is selected. The module tracks when each provider was last used and maintains a running count of successful and failed requests, giving you visibility into provider reliability and helping you decide when to rotate or replace underperforming data sources.

Team Management

The Team module allows account owners to manage user accounts and permissions. You can invite new team members by providing their name and email address, assign them the TEAM role, and configure granular permissions for each module. The permission system is module-based, meaning you can control access at the level of individual features. For each module (Dashboard, Discovery, Audit, Leads, Campaigns, Email Center, Inbox, Meetings, AI Assistant, Prompts, Team, and Settings), you can independently toggle four permissions: Can View (access the module), Can Create (add new records), Can Edit (modify existing records), and Can Delete (remove records).

Users with the OWNER role have unrestricted access to all modules and can manage team members. Users with the TEAM role can only access the modules and actions that have been explicitly granted to them. When a team member tries to access a module they do not have permission for, they see a "You don't have access to this section" message. The permission system ensures that sensitive operations like team management, system settings, and lead deletion are restricted to authorized personnel while still allowing team members to perform their day-to-day sales activities independently.

Settings

The Settings module provides system-wide configuration options organized into several categories. General settings include application name, company information, and default preferences. API configuration allows you to set up and manage the AI service endpoints and API keys that power the intelligence features. Email settings configure the SMTP server for sending outreach emails, including server address, port, authentication credentials, and sender information. Notification preferences control which events trigger notifications and through which channels. The settings are stored in the AppSetting model as key-value pairs, making them easily extensible. Any changes to settings take effect immediately and apply to all users of the system.

Lead Pipeline Stages

Every lead in TITAN progresses through a defined nine-stage sales pipeline. Understanding these stages is essential for managing your sales process effectively. Each stage represents a milestone in the prospect relationship, and the system tracks the timestamp of each transition. The stages, in order, are: DISCOVERED (initially found through lead discovery or manual entry), AUDITED (website audit has been completed), QUALIFIED (the lead meets your criteria and is worth pursuing), OUTREACH_SENT (initial contact email has been sent), REPLIED (the prospect has responded to your outreach), MEETING_BOOKED (a meeting has been scheduled with the prospect), PROPOSAL_SENT (a formal proposal or quote has been delivered), WON (the deal has been closed successfully), and LOST (the opportunity has been lost or the prospect has declined). Each stage has a corresponding color-coded badge that appears throughout the interface, making it easy to visually scan pipeline status at a glance.

Authentication and Security

TITAN uses a stateless JWT authentication system built on the jose library. When a user logs in, the server verifies their credentials against the database, generates a JWT token containing the user's ID, email, name, and role, and sets it as an httpOnly cookie. This cookie is sent automatically with every subsequent request, and the server validates it on each API call using the getSession utility function. The httpOnly flag prevents client-side JavaScript from accessing the token, providing protection against cross-site scripting attacks. Passwords are hashed using SHA-256 before storage, ensuring that plaintext passwords are never persisted in the database. The authentication system supports two user roles: OWNER (full administrative access) and TEAM (restricted access based on configured permissions).
